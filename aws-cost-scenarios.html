AWS Cost Scenarios

Tres escenarios de dimensionamiento para el pipeline de ad tracking. Cada uno tiene una arquitectura diferente, con justificación de por qué se eligió cada SKU y qué cambiaría si el tráfico crece. Región: us-east-1 · Precios on-demand · Mayo 2026.

Baja demanda
Desarrollo, staging o producto early-stage
Sistema levantado en horario de negocio o con tráfico real pero pequeño. Típico de un equipo de 2–5 personas validando el producto. El objetivo es gastar lo menos posible mientras se demuestra que la arquitectura funciona. Se usa RabbitMQ auto-gestionado en EC2 en lugar de Amazon MQ managed para evitar el costo fijo de $210/mes del broker administrado.

💡
Cómo reducirlo aún más
El NAT Gateway cuesta $33/mes solo de tarifa fija, independiente del tráfico. Para este escenario conviene usar VPC Endpoints para S3 y ECR (gratuitos o $7/mes), y sacar los servicios a una subred pública si el modelo de seguridad lo permite. Ahorro potencial: ~$26/mes. Con Savings Plan de cómputo al 1 año: reduce compute en ~22%, ahorro adicional de ~$8/mes.
⚠️
Riesgo del tier bajo: RabbitMQ self-managed en t3.medium no tiene SLA ni failover automático. Si la instancia muere, los mensajes en tránsito se pueden perder (aunque los durable queues sobreviven reinicios). Para producción real, migrar a Amazon MQ o usar un volumen EBS separado con snapshots automáticos.
Demanda media
Producto en producción con tráfico real moderado
Sistema en producción para una empresa mediana: agencia de medios, plataforma regional, o producto SaaS en crecimiento. Se introducen redundancia básica (2 réplicas del API y consumer) y Amazon MQ managed para eliminar el riesgo operativo de gestionar el broker. InfluxDB sube a m5.large para manejar el batch de escrituras cómodamente.

💡
Cómo reducirlo
Amazon MQ domina el costo con $210/mes aunque el broker esté desocupado el 80% del tiempo. Si el tráfico es predecible y el equipo tiene capacidad operativa, volver a RabbitMQ self-managed en EC2 m5.large bajaría ese ítem a ~$70/mes. Con Compute Savings Plan 1 año en ECS Fargate: ahorro de ~22% en compute (~$24/mes). Total potencial con ambas medidas: ~$370/mes.
⚠️
Punto débil: Amazon MQ mq.m5.large es single-instance — si muere, el broker no está disponible hasta que AWS lo recupera (típicamente <5 min, pero sin SLA de zero-downtime). Para HA real: usar mq.m5.large cluster ($0.576/h = $420/mes) con replicación entre AZs.
Alta demanda
Producción enterprise — load test Tier 3
Arquitectura de producción real diseñada para pasar los tiers 1, 2 y 3 del stress test. Dos réplicas de API y consumer con capacidad de escalar. InfluxDB en m5.xlarge para manejar el batch de escrituras a 1,000 rps sin degradación. Amazon MQ managed para eliminar riesgo operativo en el broker.

1,000 rps
Eventos / mes
2.6B
⚡
API
2 tasks ECS Fargate
2 vCPU / 4 GB cada task. Fastify puede manejar 2,000–5,000 rps por vCPU; 2 tasks son holgados para 1,000 rps.
📨
Queue
Amazon MQ mq.m5.large
1,000 rps de publish es trivial para un broker m5.large. El cuello de botella suele ser el consumer, no el broker.
📊
TSDB
InfluxDB en EC2 m5.xlarge
4 vCPU / 16 GB RAM. Batch de 500 puntos → 2 flushes/s a InfluxDB. Cómodo para el write path sin degradación de queries.
🪣
Object Storage
S3 Standard ~1.3 TB
Archivos horarios. Lifecycle: Glacier Instant Retrieval a los 30 días ($0.004/GB-mes vs $0.023).
🔁
Consumer
2 tasks ECS Fargate
prefetch(100) garantiza distribución sin double-counting. Puede escalar a 4 tasks para Tier 4 (2,000 rps).
📈
Dashboard
Grafana provisionado
Auto-provisionado vía docker-compose. Refresh 10s satisface el requisito de ≤60s del enunciado.

Cómo reducirlo
Con Compute Savings Plan 1 año en ECS Fargate: el compute baja de $216 a ~$168 (ahorro 22%). Mover InfluxDB a Reserved Instance m5.xlarge 1 año: $0.12/h → $87.6/mes vs $140 (ahorro $52/mes). S3 Lifecycle a Glacier a los 30 días: el costo de storage cae de $30 a ~$5/mes después del primer mes. Total potencial con estas 3 medidas: ~$620/mes.
⚠️
Para Tier 4 (2,000 rps burst): escalar consumer a 4 tasks (+$72/mes) y API a 4 tasks (+$144/mes). ECS Auto Scaling puede activar las réplicas adicionales en <60s cuando CPU supera 70%. El broker mq.m5.large aguanta 2,000 rps sin problema — el cuello de botella sería InfluxDB; considerar m5.2xlarge (+$140/mes) o sharding de escrituras.
